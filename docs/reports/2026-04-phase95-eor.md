# Phase 95 — EoR completo z=6–12

**Fecha:** 2026-04-23

## Objetivo

Integrar el módulo de reionización completo en el engine de simulación:
- Macro `maybe_reionization!` en `engine.rs` para aplicar el paso EoR en cada iteración
- Campo `uv_from_halos: bool` en `ReionizationSection` (config TOML)
- Config de prueba `configs/eor_test.toml` (N=16³, caja 10 Mpc/h, RT+reionización)
- Test de integración `phase95_eor.rs` con 6 casos

## Archivos modificados

### `crates/gadget-ng-cli/src/engine.rs`
- Nueva macro `maybe_reionization!($a_cur)`:
  - Calcula z = 1/a - 1
  - Solo actúa si `cfg.reionization.enabled && z_end ≤ z ≤ z_start`
  - Construye `n_sources` fuentes UV uniformes en la caja
  - Llama `gadget_ng_rt::reionization_step(rf, chem_scratch, sources, m1p, dt, bsz, z)`
- Llamadas agregadas en los 7 loops de stepping después de `maybe_rt!()`:
  `maybe_reionization!(a_current);`

### `crates/gadget-ng-core/src/config.rs`
- `ReionizationSection`: nuevo campo `uv_from_halos: bool` (default: false)
- `Default::default()`: `uv_from_halos: false`

### `configs/eor_test.toml` (nuevo)
Config para smoke test EoR:
```toml
[simulation]
particle_count = 4096   # 16³
box_size       = 10.0   # Mpc/h

[reionization]
enabled       = true
n_sources     = 4
uv_luminosity = 1.0
z_start       = 12.0
z_end         = 6.0
```

### `crates/gadget-ng-physics/Cargo.toml`
- Agregado `gadget-ng-rt = { path = "../gadget-ng-rt" }` como dependencia

### `crates/gadget-ng-physics/tests/phase95_eor.rs` (nuevo)
6 tests de integración:
- `reionization_state_neutral_gas`: gas neutro → x_hii_mean = 0
- `reionization_step_returns_valid_state`: paso con fuentes UV → estado coherente
- `reionization_x_hii_non_decreasing`: múltiples pasos → x_hii no disminuye
- `eor_21cm_signal_decreases_with_ionization`: δT_b disminuye con más x_HII
- `reionization_no_sources_no_change`: sin fuentes → química sin cambios
- `reionization_params_defaults`: defaults correctos (z_start=12, z_end=6)

## Tests

Todos pasan: `cargo test -p gadget-ng-physics --test phase95_eor`

## Flujo EoR en el engine

```
por cada paso de simulación:
  1. maybe_sph!(step)         → hidrodinámica SPH + enfriamiento
  2. maybe_rt!()              → solver M1 (propaga fotones)
  3. maybe_reionization!(a)   → deposita fuentes UV + actualiza x_HII
```
