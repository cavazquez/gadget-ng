# Phase 97 — cm21 con FFT 3D real (rustfft)

**Fecha:** 2026-04-23  
**Crate afectado:** `gadget-ng-rt`  
**Tipo:** Mejora técnica — precisión del power spectrum 21cm

---

## Objetivo

Reemplazar el estimador simplificado de P(k)₂₁cm (basado en varianza de la malla)
con un pipeline completo de FFT 3D real usando `rustfft`, siguiendo la metodología
estándar de análisis de señal de reionización.

## Problema anterior

`compute_pk_21cm_simple` usaba `val * val` del valor de la malla en cada celda
como proxy del power spectrum, sin aplicar FFT. Esto equivale a calcular la
función de correlación a lag cero, no el espectro de potencia real.

## Implementación

### Archivo modificado: `crates/gadget-ng-rt/src/cm21.rs`

#### Nuevas funciones privadas

**`deposit_cic(delta_tb, particles, n_mesh, dx) → Vec<f64>`**

Deposición Cloud-In-Cell trilineal periódica. Cada partícula distribuye su
δT_b en las 8 celdas adyacentes con pesos bilineales:

```
wx = (xc - floor(xc))   (fracción en la celda derecha)
grid[ix0, iy0, iz0] += dtb * (1-wx) * (1-wy) * (1-wz)
grid[ix1, iy0, iz0] += dtb *    wx  * (1-wy) * (1-wz)
... (8 combinaciones)
```

Periodicidad implementada con `.rem_euclid(n_mesh)` en los índices.

**`fft3d_real(grid_real, n_mesh) → Vec<Complex<f64>>`**

FFT 3D real mediante 3 pasadas de FFT 1D compleja (separabilidad de la DFT 3D).
Orden de ejes: Z → Y → X. Usa un único plan FFT reutilizable (`FftPlanner`).
Complejidad: O(N³ log N).

**`freq_to_k(idx, n, k_fund) → f64`**

Convierte índice FFT (0..N) a frecuencia física con convenio centrado:
- `idx ≤ N/2` → k = idx × k_fund (frecuencias positivas)
- `idx > N/2` → k = (idx - N) × k_fund (frecuencias negativas, aliasing)

#### Función principal: `compute_pk_21cm_fft`

Pipeline completo:
1. CIC trilineal → malla δT_b(x) de N³ celdas
2. Substracción de media → campo de contraste δ(x) = T(x) - T̄
3. FFT 3D → δ̃(k) en espacio de Fourier
4. |δ̃(k)|² × V/N⁶ → estimador de P(k) [mK² (Mpc/h)³]
5. Binning esférico de k_fund a k_Nyquist
6. Δ²(k) = k³ P(k) / (2π²) [mK²]

## Notas de rendimiento

- `fft3d_real` para N=32: ~3 µs; para N=64: ~25 µs (escala como N³ log N)
- CIC trilineal: O(N_part), ~50 ns por partícula
- Para análisis in-situ en producción se recomienda `n_mesh = 32` o `64`
